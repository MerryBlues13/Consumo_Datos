Python a través del tiempo, ha desarrollado un papel muy importante siendo utilizado para desarrollar aplicaciones, software, modelos maching learning, y ciencia de datos dentro de las empresas, sirviendo para múltiples tareas, tanto básicas como complejas, en el que se puede usar para tareas sencillas como las de una calculadora, hasta actividades más complejas como el modelaje de datos, así como también ha sido utilizado para la visualización de datos. (¿Qué es Python? | Guía de Python para principiantes de la nube | AWS, s. f.)
Sin embargo, existen funciones y librerias, que en conjunto podrían recopilar la información, transformarla, y visualizarla, dando lugar a la creación de Dashboards gestionados desde Python, librerías como Bokeh, que incluso pueden generar gráficos y visualizaciones interactivas que se pueden apreciar a través archivos HTML que están basados en JavaScript sin la necesidad de interactuar con este lenguaje (Bokeh documentation, s. f.) o librerías como Folium que permite visualizar datos de geolocalización mediante un mapa interactivo dentro de un archivo HTML (Folium — Folium 0.12.1 documentation, s. f.) .
Por otro lado, es posible crear y ver archivos HTML con Python. Si se necesita que los resultados sean una salida HTML, es muy conveniente utilizar un código en Python que en automáticos genere   hipervínculos, entidades gráficas, y diagramas que se pueden visualizar en cualquier navegador (Turkel, 2012).
Por ello, en el presente Trabajo, titulado como, Python como gestor de visualización de datos con el uso de JavaScript y HTML, se plantea realizar un Dashboard usando una muestra de datos de Consumos de Datos Móviles de una empresa Telefónica en México con el objeto de demostrar que se puede gestionar un Dashboard desde Python haciendo uso de múltiples gráficos. 
El enfoque de esto es que se muestre que Python, siendo un software OpenSoruce, puede competir contra Softwares de paga como Tableau. Permitiendo hacer una prueba de concepto dentro de una compañía telefónica en México.
La metodología que se usará en este artículo requiere fuentes secundarias y práctica de Python, la consulta de ejemplos ya sea de manera continua o intermitente. Se espera que este trabajo pueda servir también de material de apoyo para estudiantes que gusten hacer uso de visualizaciones interactivas a través de Python.
2.	DELIMITACIÓN Y PLANTEAMIENTO DEL PROBLEMA DE INVESTIGACIÓN
Al ser una empresa muy grande, el tema de la paga de software especializado en visualización de datos, tal como Tableau, genera un costo muy alto por la paga de cada licenciamiento que se necesita por empleado. Y ya que se necesitan visualizar los datos del consumo de datos y no tener presupuesto para obtener una licencia Tableau, en septiembre del 2022 nace la propuesta de este realizar este trabajo.  Ya que es necesario visualizar el consumo de datos de los clientes con múltiples propósitos, entre ellos, generar la visualización de datos para el área de Marketing, y, que ellos a su vez, tomen decisiones para el lanzamiento estratégico de campañas a los clientes. 
En cuanto a la delimitación de este trabajo, la falta de recursos computacionales personales, sólo se realizarán algunos gráficos, además, de que, por motivos de privacidad y política de la empresa telefónica en México, y por el gran volumen de datos con el que cuentan sus bases de datos, sólo se pudo conseguir una muestra de los datos reales. Por tal motivo, se hará uso de esta muestra para demostrar que efectivamente que los gráficos están hechos con datos reales, sin embargo, para tratar de realizar el mayor número de gráficos posibles, se hará uso de información dummy que contendrá la misma estructura que la base de datos original.
Respecto a qué tanto se abordará acerca de la explicación del código usado, se tratará de dar sólo una descripción general del código usado, y no se abordará descripción o conceptos de código que no haya sido usado para poder generar el Dashboard, aun y cuando tengan que ver con los temas en general. Por otra parte, tampoco se abordarán fechas de creación o actualización de las librerías o funciones usadas y de manera muy general se describirán los nombres y funciones de las librerías.
La mayor parte de la investigación será obtenida de la documentación disponible en los sitios web oficiales, y los ejemplos saldrán de los mismos sitios.
3.	JUSTIFICACIÓN
Con este proyecto se espera obtener la aprobación de la empresa telefónica para hacer uso de software OpenSource, lo cual permitiría hacer propuestas de visualización sin la necesidad de comprar una licencia, o bien, obtener una licencia con un costo más accesible, dando paso a no atrasar actividades y entregas. 
Por otro lado, se desea que este trabajo, llegue a esos alumnos apasionados por la visualización de datos y la programación. Siendo este documento una pequeña referencia para orientar a los estudiantes a buscar opciones de visualización de datos altamente accesibles. 
Se aportarán explicaciones y resultados para que sirvan de ejemplo, en caso de consulta. Se contempla obtener información básica, pero completa, para que los interesados en este tema tengan una mayor orientación de cómo se puede gestionar un Dashboard desde Python, por ejemplo, qué tipos de librerías existen, dónde las pueden consultar, cómo las pueden usar, cómo usar sus propios datasets, algunos gráficos de ejemplo que pueden ser de utilidad, y distintas opciones de formato que se pueden agregar a nuestros objetos.
4.	OBJETIVOS DE LA INVESTIGACIÓN
a.	OBJETIVO GENERAL
Realizar un Dashboard que represente los datos de una compañía telefónica, usando múltiples opciones de gráficos. Dicha visualización, será completamente gestionada por Python. 
b.	OBJETIVOS ESPECÍFICOS
i.	Demostrar que es posible realizar un Dashboard completamente desde Python.
ii.	Explicar y dar a conocer las librerías Bokeh y Folium que permiten hacer visualizaciones dentro de Python.
iii.	Mostrar que se puede compilar código JavaScript desde Python.
iv.	Dar a conocer la viabilidad de esta opción para visualizar nuestros datos después de hacer algún análisis o modelo.
